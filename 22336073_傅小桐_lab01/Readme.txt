收获体会：
这次个人所得税计算器项目是我第一次使用 Java 语言完成的完整项目开发。在这个过程中，我不仅巩固了 Java 的基础语法，还掌握了面向对象编程的一些核心思想，比如类的封装、模块划分和方法调用等。整个项目由五个类组成，分别是 Main、TaxView、TaxCalculator、TaxTable 以及 ChangeTable，各个类之间职责明确、逻辑清晰，保证了项目的可维护性和可拓展性。

尤其值得一提的是，这是我第一次亲自设计整个项目的接口与类的交互逻辑。起初，我花了不少时间思考如何划分各个类的职责，如何实现各类之间的数据传递与功能调用，如何既保证功能完整，又能避免冗余和耦合。通过不断尝试和调试，我逐步完成了类结构的设计，并在实践中体会到了接口设计对于项目整体架构的重要性。这种从零开始设计功能与接口的过程对我来说是一次宝贵的锻炼，不仅增强了我的逻辑思维能力，也让我对软件工程的设计原则有了更深刻的理解。

此外，我在本项目中首次尝试使用 Maven 来管理 Java 项目的依赖和构建流程，深刻体会到了构建工具的强大与便捷。通过 Maven，我能够轻松引入如 JUnit 等第三方库来辅助测试，也能通过编写 pom.xml 文件，实现项目的自动清理、编译和执行，大大提升了开发效率。

在测试方面，我学习并使用了 JUnit 框架对 TaxCalculator 类的核心功能进行了单元测试。虽然测试用例相对简单，但让我初步认识到了“回归测试”和“自动化测试”的重要性。在未来的开发中，我会继续深入学习测试技术，提高代码的健壮性和可靠性。
